Asteroids game created using C# in Visual Studio and Unity. Based on the classic Atari game Asteroids.

The game opens on a title screen, which instructs the user to press enter to begin. Randomly-placed stars zoom across the screen to give the illusion of movement.

The second game state is the opening story. Players are introduced to the characters: John, a charismatic rookie astronaut, Petrov, an old angry Russian astronaut, and Steve, an incompetent mission control worker. Steve and Petrov realize that they’re not going to Mars as they had planned, but instead, Jupiter. This means they’ll be flying through the asteroid belt!

The player takes control of the ship to fly across space and shoot or avoid the asteroids that are attempting to damage John and Petrov’s ship, the S.S. Asteroid. As the game progresses, the asteroids get faster!

The play continues until the player loses all three lives, and is sent to a game over screen, where they can compete for the highest scores.

Game controls:

  • SPACE – shoot
  • UP – accelerate
  • LEFT and RIGHT – rotate

Title and game over screen controls:

  • ENTER – progress through story/menus
  • ESC – Skip to game or go back to main menu
  • The user can also type their initials for the leaderboard

Sources:

  • Asteroid explosions, ship respawn sound, and “bloop” sound when progressing through the story were all from the asset store
  • Ship shooting sound is my friend Mo saying “pew” pitched up an octave
  • Font found here
  • Everything else was made by me

Hockey vs. Zombies is a game I made in Unity, showcasing a simulation of autonomous agents who exhibit seeking, fleeing, obstacle avoidance, and separation.

The game has two modes, simulation mode and game mode. The user can watch the autonomous agents just go about being autonomous if they want, or enter game mode and shoot some zombies. If you die in game mode, you return to simulation mode.

There are also real* NHL announcers announcing the events of the game!

Game controls:

  • SPACE – shoot
  • UP – move forward
  • LEFT and RIGHT – rotate
  • DOWN - move backward

Other on-screen buttons:

  • RESTART – restart game/simulation
  • SPAWN ZOMBIE – spawn a zombie
  • SPAWN HUMAN - spawn a human
  • HEAL HUMAN - save a boi
  • START GAME MODE - start game mode

Sources:

  • NHL Announcers
  • Human 3D Model
  • Zombie 3D Model

I created this game for the Wild MAGIC game jam at RIT on October 12th and 13th, 2019, with Justin Vaughn, Nathan Harding, and Ryan Montero

The game is a horror game, where the objective is to get through an asylum without being caught by the guards. The player has the ability to transport to an alternate dimension, where they are invisible to enemies and can walk through walls. However, the player can only be in this mode for a short amount of time before losing their sanity, so they must use their power sparingly.

This was my first time ever using Unreal, so I learned about working with Unreal blueprints through a few VFX tutorials. I made the health indicator UI and designed the second level.

Game controls:

  • WASD - Move
  • Right click - Dimension Shift
  • Left click - Interact

Take the Bait is a game developed by me and my friends Gavin DeWitt, Harry McCardell, and Ricky Norris for our game development class. We used Monogame in Visual Studio and coded in C#. I was responsible for the UI and visuals, though I worked on a lot of the back-end code too. The game is a vertical 2D-Scroller where the user has to catch as many fish as possible before hitting an obstacle.

The game is turn-based, so 2 players can go back and forth trying to out-fish each other. If a player hits an obstacle, their turn is over. Each player gets 3 turns.

The game is made more interesting with multiple levels, it gets faster and faster the longer you go without hitting an obstacle.

I learned a lot about game programming from this project, since it was one of my first big projects in any class. I learned a lot about inheritance, data structures, and all about C#, .NET, and Windows Forms.

Game controls:

  • WASD - Move

Other on-screen buttons:

  • PLAY - play the game
  • RULES - read the rules
  • SETTINGS - activates a windows form that allows you to view high scores, change hook color, and change player names.

Sources:

  • Graphics made by myself, Harry McCardell, and Food Republic

Build A Tree is a card game all about growing a tree by growing and harvesting fruit.

The game is somewhat like the card game Phase 10, in that each player needs to get a certain number of fruits on their tree in order to move to the next "phase." The player who completes three phases, as in filling their tree out with fruit, wins the game.

All players have access to mix-up cards, which can allow players to swap, steal, or change fruits in order to improve their own tree or hinder someone else's

My final project for 3D Animation and Asset Production, made with Eddie Brazier, Nick Schnering, Alex Rosenbach, and Jack Watson.

My role in the project was primarily a modeler, texturer, and animator (as was everybody else), as we split up all the 3D models in the scene evenly. I made the snowglobe the user walks around in, the bookshelf and books, the mugs, and the snowman that looks like Frosty the Snowman. The snowman is animated in Maya, his hat falls off when the sledding snowman runs into him.

In addition to my assigned role as a modeler, I worked a lot to keep the git maintained and up to date. I helped my teammates learn how to use git, and even though it can be a bit of a pain it was so much easier than sending back-and-forth over Google Drive or something like that.
